PNK Inmobiliaria en GitHub Pages

1

Crear Repositorio en GitHub

Ve a GitHub.com e inicia sesión con tu cuenta

Haz clic en: Botón + (arriba a la derecha) → New repository

Rellena los campos:

  • Repository name: pnk-inmobiliaria
  • Description: "Sitio web PNK Inmobiliaria"
  • Visibility: Selecciona "Public"
  • Initialize: Deja sin marcar
✓ Haz clic en "Create repository"
2

Preparar Git en tu Computadora

Si no tienes Git instalado, descárgalo desde git-scm.com

Abre la Terminal/CMD en la carpeta proyecto:

# Configura tu usuario (solo la primera vez) git config --global user.name "Tu Nombre" git config --global user.email "tu-email@example.com"
💡 Usa el nombre y email que registraste en GitHub
3

Subir Archivos a GitHub (Comandos Principales)

Copia y pega estos comandos en la terminal (carpeta proyecto):

git init git add . git commit -m "Inicial: PNK Inmobiliaria" git branch -M main git remote add origin https://github.com/TU-USUARIO/pnk-inmobiliaria.git git push -u origin main
⚠️ IMPORTANTE: Reemplaza "TU-USUARIO" con tu nombre de usuario de GitHub
✓ Si todo sale bien verás: "Enumerating objects... remote: Counting objects..." y luego "done"
4

Activar GitHub Pages

Vuelve a GitHub en el navegador

1. Haz clic en Settings (Configuración)
2. En el menú izquierdo, busca Pages
3. Bajo "Source", selecciona: "Deploy from a branch"
4. Rama: "main"
5. Carpeta: "/ (root)"
6. Haz clic en Save
5

Esperar y Verificar

GitHub tardará algunos segundos en desplegar tu sitio

Vuelve a la página principal del repositorio. Verás un mensaje verde que dice:
"Your site is published at..."

Copia ese enlace y ábrelo en tu navegador

🎉 ¡Tu sitio está en línea!

🔧 Problemas Comunes y Soluciones

❌ Error: "fatal: not a git repository"

Solución: Asegúrate de estar en la carpeta proyecto

cd C:\ruta\a\tu\proyecto

❌ Error: "remote already exists"

Solución: Elimina el remote anterior y agrega el correcto

git remote remove origin git remote add origin https://github.com/TU-USUARIO/pnk-inmobiliaria.git git push -u origin main

❌ El sitio no aparece en línea

Soluciones:

  • Espera 5-10 minutos adicionales
  • Limpia el caché: Ctrl+Shift+Delete en el navegador
  • Verifica que GitHub Pages esté habilitado
  • Comprueba que el commit llegó a GitHub (mira el código en el navegador)

❌ Error de autenticación con GitHub

Solución: Usa un token de acceso personal

  1. Ve a github.com/settings/tokens
  2. Crea un nuevo token (scope: repo)
  3. Usa el token como contraseña en Git
